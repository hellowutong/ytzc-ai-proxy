## 一、产品概述

### 1.1 产品名称
**TW AI节能器** - 本地可部署的 AI 代理中枢

### 1.2 核心价值
为个人开发者提供轻量级、本地部署的 AI 代理服务，实现多模型统一调度、成本优化、会话沉淀与 Skill 自动化生成。

### 1.3 目标场景
- 个人开发者本地使用
- 跨平台：Windows / Linux / macOS
- 当前仅本地使用，未来支持公网暴露

## 五、代理连接管理

### 5.1 连接 Key 规则
- **生成策略**：每组（小模型+大模型组合）只产生一个 Key
- **Key 格式**：`tw-{随机字符串}`
- **唯一性**：Key 全局唯一

### 5.2 连接管理功能

| 功能 | 描述 |
|------|------|
| 查看连接 | UI 展示连接名称、Key（脱敏）、状态 |
| 禁用连接 | 可禁用某个连接 Key |
| 删除连接 | 可删除某个连接 Key |
| 调用 BaseURL | 展示调用路径 `https://host:port/proxy/v1/` |

### 5.3 密钥安全
- **存储**：API Key 存储在 config.yml
- **日志**：密钥不落日志
- **UI 显示**：Key 只显示前后各 4 位，如 `tw-a1b2****3c4d`

## 六、智能调度策略

### 6.1 调度流程

```
用户请求 → 验证 Proxy Key → 小模型处理 → 超时？
                                              │
                                              ├── 否 → 返回结果
                                              │
                                              ├── 是 → 失败次数+1
                                              │            │
                                              │         连续 N 次？
                                              │            │
                                              ├── 否 → 重试小模型
                                              │
                                              └── 是 → 切换大模型处理
```

### 6.2 失败判定规则

| 规则 | 说明 |
|------|------|
| 失败定义 | 请求超时（超过配置的 timeout_ms） |
| 失败计数 | 按 Proxy Key 统计连续失败次数 |
| 切换阈值 | 连续失败 N 次后自动切换大模型 |
| 默认阈值 | N = 3 次（config.yml 可配置） |

### 6.3 用户无感切换
- 切换过程对调用方完全透明
- 调用方无需感知小模型/大模型切换
- 最终只返回大模型的响应结果

## 七、会话管理

### 7.1 会话定义
- **会话范围**：同一个 Proxy Key 的请求归为一个会话
- **会话隔离**：不同 Key 的会话相互独立

### 7.2 会话结束判定

| 触发方式 | 说明 |
|----------|------|
| 客户端结束 | 客户端主动发送结束信号 |
| 超时结束 | 超过 config.yml 中 `session_timeout` 无活动自动结束 |

### 7.3 会话存储

| 存储位置 | 内容 |
|----------|------|
| MongoDB | 会话消息、摘要、元数据 |
| Qdrant | 会话摘要向量（用于后续检索增强，预留） |

### 7.4 会话功能

| 功能 | 描述 | MVP |
|------|------|-----|
| 自动总结 | 会话结束时自动生成摘要 | ✅ |
| 摘要存储 | 摘要写入 MongoDB | ✅ |
| 向量化 | 摘要向量化存入 Qdrant | ✅ |
| 历史列表 | UI 展示历史会话列表 | ✅ |
| 查看详情 | UI 查看会话详情 | ✅ |
| 原始记录 | UI 上可选显示（默认隐藏，点击显示） | ✅ |
| 删除历史 | 物理删除（不可恢复） | ✅ |
| 批量删除 | 支持批量删除会话 | ✅ |

### 7.5 会话数据结构

## 九、向量存储（Qdrant）

### 9.1 向量类型

| 向量类型 | 用途 | 集合名 |
|----------|------|--------|
| 会话摘要向量 | 会话内容向量化，用于相似检索 | `sessions` |
| Skill 向量 | Skill 内容向量化，用于匹配调用 | `skills` |

### 9.2 向量操作

| 操作 | 说明 |
|------|------|
| 添加 | 会话结束/ Skill 发布时自动向量化 |
| 搜索 | 支持相似度检索，分页返回 |
| 更新 | 修改内容后重建向量 |
| 删除 | 物理删除向量点 |
| 重建 | 支持重建单个或全部向量索引 |

### 9.3 相似度阈值
- 默认阈值：score < 0.8 不匹配
- 后续版本用于检索增强

## 十、配置管理

### 10.1 配置项

```yaml
# config.yml
app:
  host: "0.0.0.0"
  port: 8080
  data_path: "./data"
  session_timeout: 1800  # 会话超时（秒）
  model_timeout: 30000   # 模型调用超时（毫秒）

strategy:
  fail_threshold: 3      # 连续失败次数阈值
  small_model_retry: 0   # 小模型重试次数（0=不重试）

providers:
  - name: "xxx"
    small_model:
      name: "xxx"
      base_url: "xxx"
      api_key: "xxx"
    big_model:
      name: "xxx"
      base_url: "xxx"
      api_key: "xxx"
    proxy_key: "tw-xxx"  # 自动生成
```

### 10.2 配置生效
- 修改 `config.yml` 后，Web UI 点击"应用"按钮重新加载配置
- 或调用后台 API 触发重新加载

## 十七、Skill 格式规范（CC 兼容）

本系统 Skill 格式兼容 **Cursor CC Agent Skills** 开放标准。

### 17.1 目录结构

```
skill-name/
├── SKILL.md          # 必需：技能主文件（含 YAML frontmatter）
├── scripts/          # 可选：可执行代码
├── references/       # 可选：附加文档
└── assets/           # 可选：静态资源（模板、图片等）
```

### 17.2 SKILL.md 文件格式

每个 Skill 在 `SKILL.md` 文件中定义，包含 **YAML Frontmatter** 和 **Markdown 主体**：

```markdown
---
name: skill-xxxx           # 必需：技能标识符（小写字母、数字、连字符）
description: 简要描述此技能的功能及使用时机。  # 必需：供 Agent 判断相关性
license: MIT               # 可选：许可证名称
compatibility:             # 可选：运行环境要求
metadata:                  # 可选：额外元数据
  author: xxx
  version: 1.0
---

# 技能名称

为 Agent 提供的详细指令。

## 使用时机

- 在以下情况使用此技能...
- 此技能适用于...

## 指令

- 为 Agent 提供的分步指导
- 特定领域的约定
- 最佳实践和模式
```

### 17.3 Frontmatter 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | ✅ | 技能标识符，仅限小写字母、数字和连字符，必须与父文件夹名称一致 |
| `description` | ✅ | 描述技能的作用及其使用场景，由 Agent 用于判断相关性 |
| `license` | ❌ | 许可证名称或对随附许可证文件的引用 |
| `compatibility` | ❌ | 运行环境要求（系统软件包、网络访问等） |
| `metadata` | ❌ | 用于额外元数据的任意键值映射 |

### 17.4 可选目录说明

| 目录 | 用途 |
|------|------|
| `scripts/` | Agent 可以运行的可执行代码（Bash、Python、JavaScript 等） |
| `references/` | 按需加载的附加文档（详细参考资料、API 文档等） |
| `assets/` | 模板、图片或数据文件等静态资源 |

### 17.5 本系统扩展字段

在 CC 规范基础上，本系统增加以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `base_id` | string | Skill 的稳定标识（系统生成） |
| `version_id` | integer | 版本号，从 0 开始递增 |
| `status` | string | 状态：draft/published/deprecated |
| `source_session_id` | string | 来源会话 ID（用于溯源） |

### 17.6 Skill 示例

```markdown
---
name: python-code-generator
description: 生成高质量 Python 代码。用于用户请求编写、修复或优化 Python 代码时。
metadata:
  author: system
  created_at: 2026-02-02
---

# Python Code Generator

为 Agent 提供生成高质量 Python 代码的指导。

## 使用时机

- 用户请求编写新功能时
- 用户请求修复 Bug 时
- 用户请求优化现有代码时

## 编码规范

- 遵循 PEP 8 规范
- 使用类型注解
- 编写单元测试
```

## 十八、大模型结果触发 Skill 迭代流程

### 18.1 触发机制

| 触发方式 | 说明 |
|----------|------|
| **自动触发** | 小模型连续失败 N 次后，切换大模型处理完成时，自动生成 Skill 草稿 |
| **手动触发** | 用户在历史会话中手动选择"生成 Skill" |

### 18.2 生成流程

```
小模型连续失败 N 次 → 切换大模型处理 → 处理完成
                                          ↓
                              自动生成 Skill 草稿（draft）
                                          ↓
                              UI 展示：来源标注
                                          ↓
                              用户审核 → 确认发布 / 编辑后发布
```

### 18.3 草稿属性

| 属性 | 值 |
|------|-----|
| 状态 | `draft`（草稿，需审核确认） |
| 来源标注 | 记录来源会话 ID、触发原因（连续失败 N 次） |

### 18.4 来源标注示例

```json
{
  "source_session_id": "session-uuid-xxxx",
  "trigger_reason": "小模型连续失败 3 次，切换大模型处理",
  "generated_at": "2026-02-02T12:00:00Z"
}
```

## 十九、导入 ZIP 格式规范

### 19.1 ZIP 包结构

```
skill-export.zip
├── skill.md          # Skill 主文件（必需）
├── examples/         # 可选：示例目录
│   └── example1.md
├── assets/           # 可选：资源目录
│   └── diagram.png
└── metadata.json     # 元数据文件（基础信息）
```

### 19.2 metadata.json 字段（基础信息）

```json
{
  "base_id": "skill-xxxx",
  "version_id": 0,
  "source_session_id": "session-xxx",
  "change_reason": "initial",
  "imported_at": "2026-02-02T12:00:00Z"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `base_id` | string | Skill 稳定标识 |
| `version_id` | integer | 版本号 |
| `source_session_id` | string | 来源会话 ID |
| `change_reason` | string | 变更原因 |
| `imported_at` | string | 导入时间 |

### 19.3 版本处理规则

| 场景 | 处理方式 |
|------|----------|
| 版本号相同 | 覆盖现有版本 |
| 版本号不同 | 创建新版本 |

### 19.4 冲突处理策略

导入时遇到文件冲突，用户可选择：
- A) **跳过冲突文件** - 不导入冲突文件，保持现有文件
- B) **全部覆盖** - 用 ZIP 中的文件替换现有文件
- C) **保留两者** - 重命名后导入（如果支持）

## 二十、baseskill 模板管理

### 20.1 baseskill 概述

baseskill 是系统内置的基础技能模板，用于定义系统核心流程的生成规则。

### 20.2 模板类型

| # | 模板名称 | 用途 | 调用环节 |
|---|----------|------|----------|
| 1 | session-summarizer | 会话摘要生成 | 会话结束时 |
| 2 | skill-generator | Skill 草稿生成 | 生成 Skill 时 |
| 3 | skill-name-generator | Skill 名称生成 | 生成 Skill 时 |
| 4 | vectorizer | 向量化处理 | Skill 发布后 |

### 20.3 模板文件结构

每个 baseskill 模板包含以下文件：

```
baseskill/
├── skill-generator/          # Skill 生成器模板
│   ├── SKILL.md              # 技能定义
│   ├── prompt-template.md    # Prompt 模板
│   └── rules.yaml            # 生成规则配置
│
├── session-summarizer/       # 会话摘要模板
│   ├── SKILL.md
│   ├── prompt-template.md
│   └── rules.yaml
│
├── vectorizer/               # 向量化模板
│   ├── SKILL.md
│   ├── prompt-template.md
│   └── rules.yaml
│
└── skill-name-generator/     # Skill 名称生成模板
    ├── SKILL.md
    ├── prompt-template.md
    └── rules.yaml
```

### 20.4 模板调用流程

#### 启动阶段

```
系统启动
    │
    ├── 加载 config.yml
    ├── 扫描 baseskill/ 目录（系统内置，只读）
    └── 扫描 data/baseskill/ 目录（用户自定义，可写）
```

#### 运行时阶段

| 模板 | 触发时机 | 调用内容 |
|------|----------|----------|
| session-summarizer | 会话结束 | 加载 rules.yaml → 执行摘要生成规则 |
| skill-generator | 生成 Skill 草稿时 | 加载 rules.yaml → 执行 Skill 生成规则 |
| skill-name-generator | 生成 Skill 时 | 加载 rules.yaml → 执行名称生成规则 |
| vectorizer | Skill 发布后 | 加载 rules.yaml → 执行向量化规则 |

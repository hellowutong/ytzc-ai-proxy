## 一、代理请求处理流程

### 1.1 完整请求流程

```mermaid
sequenceDiagram
    participant Client as 调用方
    participant Proxy as Proxy API
    participant Auth as 认证中间件
    participant ProxySvc as ProxyService
    participant Small as 小模型
    participant Big as 大模型
    participant MongoDB as MongoDB
    participant Qdrant as Qdrant

    Client->>Proxy: POST /proxy/v1/chat/completions
    Proxy->>Auth: 验证 Proxy Key
    
    alt Key 无效
        Auth-->>Client: 401 Unauthorized
    else Key 有效
        Auth->>Proxy: 返回连接配置
        
        Proxy->>ProxySvc: 转发请求
        ProxySvc->>ProxySvc: 获取连接配置
        
        Note over ProxySvc: 开始小模型调用
        ProxySvc->>Small: 调用小模型
        
        alt 小模型成功
            Small-->>ProxySvc: 返回响应
            ProxySvc->>MongoDB: 记录会话
        else 小模型超时
            Small-->>ProxySvc: Timeout
            
            ProxySvc->>ProxySvc: fail_count + 1
            
            alt fail_count >= N
                Note over ProxySvc: 切换大模型
                ProxySvc->>Big: 调用大模型
                Big-->>ProxySvc: 返回响应
                ProxySvc->>MongoDB: 记录会话
                ProxySvc->>ProxySvc: 自动生成 Skill 草稿
                ProxySvc->>Qdrant: 向量化
            else
                ProxySvc->>Small: 重试小模型
            end
        end
        
        ProxySvc-->>Client: 返回响应
    end
```

### 1.2 调度策略流程图

```mermaid
flowchart TD
    A[用户请求] --> B[验证 Proxy Key]
    
    B --> C{Key 有效?}
    C -->|否| D[返回 401]
    C -->|是| E[获取连接配置]
    
    E --> F[调用小模型]
    
    F --> G{成功?}
    G -->|是| H[返回结果]
    G -->|否| I[超时?]
    
    I -->|否| J[返回错误]
    I -->|是| K[fail_count + 1]
    
    K --> L{fail_count >= N?}
    L -->|否| F
    L -->|是| M[fail_count = 0]
    
    M --> N[调用大模型]
    N --> O[返回结果]
    
    H --> P[记录会话]
    O --> P
    
    P --> Q[会话结束?]
    
    Q -->|是| R[生成摘要]
    Q -->|否| S[等待更多消息]
    
    R --> T[向量化]
    T --> U[生成 Skill 草稿?]
    
    U -->|是| V[保存草稿]
    U -->|否| W[结束]
    
    V --> W
```

## 二、会话管理流程

### 2.1 会话生命周期

```mermaid
stateDiagram-v2
    [*] --> Active: 创建会话
    
    Active --> Active: 接收消息
    Active --> Ended: 手动结束
    Active --> Ended: 超时自动结束
    
    Ended --> [*]: 清理资源
    
    note right of Active
        状态: active
        记录消息
        可生成摘要
    end note
    
    note right of Ended
        状态: ended
        已生成摘要
        已向量化
    end note
```

### 2.2 会话 CRUD 流程

```mermaid
flowchart LR
    subgraph Create [创建]
        A1[客户端请求] --> A2[创建会话]
        A2 --> A3[生成 session_id]
        A3 --> A4[保存到 MongoDB]
    end
    
    subgraph Read [读取]
        B1[列表查询] --> B2[分页筛选]
        B2 --> B3[返回列表]
        
        B4[详情查询] --> B5[按 ID 查找]
        B5 --> B6[返回详情]
    end
    
    subgraph Update [更新]
        C1[更新请求] --> C2[验证权限]
        C2 --> C3[更新数据]
        C3 --> C4[保存变更]
    end
    
    subgraph Delete [删除]
        D1[删除请求] --> D2[物理删除]
        D2 --> D3[删除关联向量]
        D3 --> D4[返回结果]
    end
```

## 三、Skill 管理流程

### 3.1 Skill 生成流程

```mermaid
flowchart TD
    A[选择会话] --> B[点击生成 Skill]
    
    B --> C[提取会话内容]
    C --> D[调用 LLM 生成草稿]
    
    D --> E[保存为 draft]
    E --> F[UI 展示草稿]
    
    F --> G{编辑草稿?}
    G -->|是| H[编辑内容]
    H --> F
    G -->|否| I[确认发布]
    
    I --> J[状态变为 published]
    J --> K[更新 active_version]
    K --> L[旧版本标记 deprecated]
    L --> M[向量化]
    M --> N[结束]
```

### 3.2 Skill 版本管理

```mermaid
flowchart TD
    A[Skill] --> B[创建新版本]
    B --> C[版本号 + 1]
    C --> D[状态: draft]
    
    D --> E{审核确认?}
    E -->|否| F[继续编辑]
    F --> D
    E -->|是| G[发布]
    
    G --> H[状态: published]
    H --> I[设置 active_version]
    I --> J[旧版本 deprecated]
    
    J --> K{需要回滚?}
    K -->|是| L[选择目标版本]
    L --> M[切换 active_version]
    M --> N[目标版本 published]
    N --> O[当前版本 deprecated]
    O --> P[重新向量化]
    K -->|否| Q[结束]
    
    P --> Q
```

### 3.3 Skill 导入流程

```mermaid
flowchart TD
    A[上传 ZIP 文件] --> B[解压文件]
    B --> C[验证结构]
    
    C --> D{结构有效?}
    D -->|否| E[返回错误]
    D -->|是| F[读取 metadata.json]
    
    F --> G{版本存在?}
    G -->|是| H{冲突处理策略?}
    
    H -->|跳过| I[跳过冲突文件]
    H -->|覆盖| J[覆盖现有文件]
    H -->|保留两者| K[重命名后导入]
    
    G -->|否| L[创建新版本]
    
    I --> M[导入完成]
    J --> M
    K --> M
    L --> M
    
    M --> N[向量化]
    N --> O[更新索引]
    O --> P[结束]
```

### 3.4 Skill 导出流程

```mermaid
flowchart TD
    A[选择 Skill] --> B[选择版本]
    B --> C{批量导出?}
    
    C -->|是| D[收集所有版本]
    C -->|否| E[只选当前版本]
    
    D --> F[打包文件]
    E --> F
    
    F --> G[添加 metadata.json]
    G --> H[压缩为 ZIP]
    H --> I[返回下载链接]
    I --> J[结束]
```

## 八、整体架构数据流

```mermaid
flowchart TB
    subgraph External [外部系统]
        Client[调用方]
        Provider[上游供应商]
    end
    
    subgraph ProxySystem [TW AI 节能器]
        direction TB
        
        subgraph API [API 层]
            ProxyAPI[/proxy/v1/*]
            AdminAPI[/api/v1/*]
            WebAPI[/web/v1/*]
        end
        
        subgraph Service [服务层]
            ProxyService[代理服务]
            SessionService[会话服务]
            SkillService[Skill 服务]
            VectorService[向量服务]
            ConfigService[配置服务]
        end
        
        subgraph Domain [领域层]
            ProxyDomain[代理网关]
            SessionDomain[会话管理]
            SkillDomain[Skill 管理]
            VectorDomain[向量存储]
        end
        
        subgraph Infra [基础设施层]
            MongoDB[MongoDB]
            Qdrant[Qdrant]
            Files[文件系统]
            Baseskill[baseskill 模板]
        end
    end
    
    Client --> ProxyAPI
    ProxyAPI --> ProxyService
    ProxyService --> ProxyDomain
    ProxyService --> Provider
    
    AdminAPI --> SessionService
    AdminAPI --> SkillService
    AdminAPI --> VectorService
    AdminAPI --> ConfigService
    
    SessionService --> SessionDomain
    SkillService --> SkillDomain
    VectorService --> VectorDomain
    
    SessionDomain --> MongoDB
    SessionDomain --> Qdrant
    SkillDomain --> Files
    SkillDomain --> Qdrant
    VectorDomain --> Qdrant
    
    SessionService --> Baseskill
    SkillService --> Baseskill
```
